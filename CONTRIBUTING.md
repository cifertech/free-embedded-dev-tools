# Contributing

Thanks for helping keep this list accurate and useful. A few guidelines before opening a PR:

## Adding a new entry

- One row per tool/service, added to the most relevant existing table. If nothing fits, propose a new section.
- Format: `| [Name](https://link) | What's actually free | Real limits / what it's best for |`
- Link to the tool's own site or official docs, not a reseller/affiliate page.
- Keep the description factual and specific. "Great tool!" isn't useful — "Free tier caps at 100 device connections" is.

## What counts as "free"

- Genuinely no-cost, ongoing access — not a 14-day trial, not "free" that requires a credit card and auto-charges after a window.
- If something is free only under specific conditions (non-commercial use, open-source projects only, education email required), **say so explicitly** in the Notes column. Don't let someone find out the hard way mid-project.
- Freemium tools are fine as long as the free tier is genuinely usable on its own, not just a crippled demo.

## Keeping entries accurate

- Free tiers and limits change without notice. If you notice a stale entry (dead link, changed pricing, discontinued service), open a PR to fix or remove it — small accuracy PRs are just as welcome as new entries.
- If a tool gets acquired, shuts down, or changes its free tier significantly, please update or flag it rather than leaving it stale for the next reader.

## Style

- Keep rows sorted roughly by how commonly useful the tool is within its section (most broadly useful first), not alphabetically.
- Don't add promotional language, referral links, or affiliate codes.
- Don't add anything that requires the person to sign up for something before they can tell if it's actually useful to them, unless there's genuinely no way around it (call this out in the notes).

## Scope

This list is specifically for **embedded / firmware / hardware development** — if a tool is really a general-purpose dev tool with no particular embedded angle, it probably belongs in a broader list like `free-for-dev` instead, not here.

That's it — open a PR and it'll get reviewed. Thanks for contributing.
